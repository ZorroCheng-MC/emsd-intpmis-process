# System Installation Plan for I&T Project Management Information System (INTPMIS)

This document outlines the system installation plan for the I&T Project Management Information System (INTPMIS) developed for the Electrical and Mechanical Services Department (EMSD). It consolidates information from the provided User Requirement Specification (URS) and other documentation to provide a comprehensive overview of the installation process.

## 1. System Overview

The INTPMIS aims to address the limitations of the current system by providing an easy-to-use web portal for managing I&T projects, wishes, and solutions. Key features include:

*   **Working Environment:** A web-based portal with user-friendly UI and forms for administrators and users.
*   **Data Management:** Storage of I&T project information (including E&M InnoPortal and other I&T projects) in a MySQL database.  Supports text, data, reports, and attachments (PDF, Microsoft Word, JPEG, PNG, etc.).  Includes data migration from existing systems and Excel tables.
*   **Data Analysis:** Generation of reports and statistics to analyze I&T development trends.  Customizable reports, filtering, sorting, and data export to Excel.
*   **Workflow:** Automated data retrieval from different tables when creating new projects, reducing manual data entry and improving accuracy.

## 2. System Functions

The INTPMIS provides the following system functions:

*   **Information Viewing:**
    *   View Project List and Detail
    *   View Wish List and Detail
    *   View Solution List and Detail
    *   View Report List and Detail
    *   View Log List and Detail
    *   Search Projects, Wishes, and Solutions
    *   Filter Projects, Wishes, and Solutions
*   **Content Management:**
    *   Create a Project
    *   Edit a Project
    *   Edit a Wish
    *   Edit a Solution
    *   Cancel Reminder for Log
*   **Workflow Application:**
    *   Export Projects
    *   Export Wishes
    *   Export Solutions
    *   Export Report
    *   Upload Files

## 3. Functional Requirements

The system must fulfill the following functional requirements:

*   **REQ-SYS-001 - Login:**
    *   Support single sign-on through Windows Active Directory.
    *   Support existing user permission matrix mapping.
    *   Provide an option for users acting on behalf of others to access the system under the acting role or their original role.
*   **REQ-SYS-002 - Project List:**
    *   Allow users to search for projects.
    *   Support sorting and filtering.
    *   Allow exporting the project list to a CSV file.
    *   Allow configuring filtering and sorting criteria.
    *   Allow viewing and editing projects.
*   **REQ-SYS-003 - Create a Project:**
    *   Allow users to create a project.
*   **REQ-SYS-004 - View and Edit a Project:**
    *   Support read-only and edit modes based on user permissions.
*   **REQ-SYS-005 - Wish List:**
    *   Allow users to search for wishes.
    *   Support sorting and filtering.
    *   Allow exporting the wish list to a CSV file.
    *   Allow configuring filtering and sorting criteria.
    *   Allow viewing and editing wishes.
*   **REQ-SYS-006 - View or Edit a Wish:**
    *   Support read-only and edit modes based on user permissions.
*   **REQ-SYS-007 - Solution List:**
    *   Allow users to search for solutions.
    *   Support sorting and filtering.
    *   Allow exporting the solution list to a CSV file.
    *   Allow configuring filtering and sorting criteria.
    *   Allow viewing and editing solutions.
*   **REQ-SYS-008 - View and Edit a Solution:**
    *   Support read-only and edit modes based on user permissions.
*   **REQ-SYS-009 - Report List:**
    *   Support sorting and filtering.
    *   Allow exporting the report list to a CSV file.
    *   Allow configuring filtering and sorting criteria.
*   **REQ-SYS-010 - View Report:**
    *   Support read-only mode based on user permissions.
*   **REQ-SYS-011 - Log List:**
    *   Support sorting and filtering.
    *   Allow exporting the log list to a CSV file.
    *   Allow configuring filtering and sorting criteria.
    *   Allow viewing and editing logs.
*   **REQ-SYS-012 - View Log List:**
    *   Support read-only mode based on user permissions.
*   **REQ-SYS-013 - Upload Files:**
    *   Allow users to upload multiple file formats (PDF, JPEG, DOCX, XLS, etc.).
    *   Allow users to replace files.
    *   Allow users to delete files.

## 4. Non-Functional Requirements

The system must meet the following non-functional requirements:

*   **REQ-SR-001 - Response Time:** 95% of predefined queries should respond within 5 seconds.
*   **REQ-SR-002 - Service Time:**
    *   System access from 08:00 to 20:00 daily (late night provisionally). 7x24 availability is not required.
    *   Availability of 99.0% (unscheduled downtime not exceeding 7.2 minutes per day or 1.83 days per year on average).
*   **REQ-SR-003 - Export:** Allow data to be exported in easy-readable formats like Excel.
*   **REQ-SR-004 - Scalability:** Provide scalability options for future system expansion.
*   **REQ-SR-005 - System Performance:** Provide a technically sound architecture to meet response time and performance levels.
*   **REQ-UR-001 - User Interface Requirement:**
    *   User-friendly interfaces and GUI. Personalized views based on user profiles. Consistent with web-based applications.
    *   Responsive web design for different screen sizes (desktops, tablets, mobile phones).
    *   Function and display properly in multiple browsers (Internet Explorer, Chrome with Apollo support).
*   **REQ-SCR-001 - Function Access Rights:** Define user access rights based on operations or user levels.
*   **REQ-SCR-002 - Access to Data:**
    *   Define access rights for users to access particular data, conditionally.
    *   Define access monitoring for selected data.
*   **REQ-SCR-003 - System Backup:** Meet general backup requirements, including data, system, and audit logging information.

## 5. Technical Requirements

The system must meet the following technical requirements:

*   **TR-SBR-001 - Server House Keeping:** Archive system logs to backup tape weekly.
*   **TR-SBR-002 - Backup, Recovery and System Archive:**
    *   Provide daily backup.
    *   Provide weekly system backup.
    *   Provide 2 generations of system backup stored off-site.
    *   Provide system activity logs and reports.
*   **TR-DRR-001 - System Disaster Recovery:**
    *   Participate in disaster recovery activities.
    *   Coordinate with new system users to participate in disaster recovery activities.
    *   Resume the whole system from the disaster recovery site to the normal production environment.

## 6. Installation Steps

While a detailed installation guide is not provided, the following steps can be inferred:

1.  **Server Setup:** Configure the server environment to meet the system's technical requirements. This includes installing the necessary operating system, web server, database server (MySQL), and other dependencies.
2.  **Database Setup:** Create the MySQL database and import the initial database schema.  Migrate existing I&T project information from other systems and Excel tables.
3.  **Application Deployment:** Deploy the INTPMIS application code to the web server.
4.  **Configuration:** Configure the application to connect to the database and other services.
5.  **User Account Setup:** Create user accounts and assign appropriate roles and permissions based on LDAP login details and existing permission matrix mapping.
6.  **Testing:** Thoroughly test the system to ensure that all functional and non-functional requirements are met.
7.  **Training:** Provide training to users on how to use the system.
8.  **Go-Live:** Deploy the system to the production environment.
9.  **Monitoring:** Monitor the system to ensure that it is performing as expected.

## 7. User Procedures and Operation Manual

The following documentation should be provided to users:

*   Training courses and user experience sessions.
*   Program Manual including workflow manual.
*   System manual.
*   User procedures manual.

## 8. Key Features and Functionality

*   **Search Function:**  Users can search for projects, wishes, or solutions by selecting one or more search criteria (fields of the object). The system displays possible values for the selected fields, allowing users to refine their search.
*   **Export Function:** Users can export all records after a search to a CSV file.
*   **Printing Report Function:** Users can print reports by clicking the "Print" button. A preview page allows users to adjust printing settings.
*   **Logging:** The system logs all create and edit actions on projects, wishes, and solutions.  Users can view log records, filter by tag type, and cancel reminders for specific records.

## 9. Administrator Procedures

*   **Project Management:** Administrators have enhanced editing capabilities for projects, including fields like "Date of Uploading Interim Evaluation Report."
*   **Log Management:** Administrators can view and export log records, cancel reminders for individual or multiple log entries.
